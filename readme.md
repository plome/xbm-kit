# xbm-kit

XBM 十六进制数组与位图图像双向转换的命令行工具，专为 **u8g2**、**LVGL** 等嵌入式显示库设计。

## 功能特点

- **编码**：将 PNG / BMP / JPG 等图片转换为 XBM 十六进制数组，直接生成带 `PROGMEM` 的 C 数组模板（含图片尺寸注释）
- **解码**：从 C 代码中粘贴 XBM 数组，渲染为图像并预览，可保存为 BMP 文件
- **剪贴板集成**：自动复制生成的数组或完整模板，无需手动框选
- **本地运行**：所有转换在本地完成，不上传任何数据，保护项目隐私
- **无过度设计**：阈值固定 128，位顺序固定 LSB，贴合常见嵌入式库的默认行为

## 使用方法

运行脚本：

```bash
python xbm-kit.py
```

### 编码模式（图片 → XBM 数组）

1. 将图片文件拖入终端（或手动输入路径），按回车
2. 输入数组名称（如 `my_icon`），或直接回车仅生成数组内容
3. 生成的 C 数组模板将自动复制到剪贴板，并显示在终端

示例输出：

```c
static const uint8_t my_icon[] PROGMEM =      //my_icon，32*32
{
  0xFF, 0x00, 0x81, ...
};
```

### 解码模式（XBM 数组 → 图片）

1. 输入图片宽度（数字），回车
2. 输入图片高度，回车
3. 粘贴 XBM 数组内容（支持多行），然后按一次空行结束
4. 脚本将显示图像预览，并询问是否保存为 BMP 文件

## 示例

### 编码：将 logo.png 转为 XBM 数组

```
请输入图像宽度【数字】或【拖入图片文件】: logo.png
图片尺寸: 64 x 64，已编码完毕。

请输入数组名称（直接按回车仅复制数组内容）: logo

生成的模板数组:
static const uint8_t logo[] PROGMEM =      //logo_64x64，64*64
{
  0x00, 0x01, ...
};
已复制完整模板到剪贴板。
```

### 解码：从现有代码中查看图片

```
请输入图像宽度【数字】或【拖入图片文件】: 32
请输入图像高度: 32

请点击【鼠标右键】粘贴XBM数组数据:
格式: {0x00, 0x01, 0x02, ...}
粘贴后按回车继续:
0xFF,0x81,0x00,   （此处粘贴数组内容）
（再按一次空行）

解析到 128 字节数据，正在预览...
是否保存为BMP文件？(y/n):
```
